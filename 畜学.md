# 畜学（ちくがく） ─ 畜物語のつくりかた教本

> **このファイルは「作業前にまず読む」教本です。**
> 畜物語のゲームやコンテンツを **作る／直す前に、必ず最初にこの畜学を読んでから着手する**こと。
> ここには共通作法・必須要素・中毒設計・公開ワークフロー・過去の失敗からの教訓がまとまっている。
> （更新の履歴＝「畜史」は `changelog/index.html`。畜史は“歴史”、畜学は“作法”。役割が違う。）

---

## 0. 畜物語とは
- 絵文字の島でのんびり遊ぶ、**こっこちゃん**のブラウザゲーム集（chikumo.jp）。
- 各ゲームは `<ディレクトリ>/index.html` に **HTML＋インラインJS＋Canvas/WebGL** で**1ファイル自己完結**。
- 外部依存は最小：**公式SVG `/chikumo.svg`** と、ランキング用の **Firebase compat スクリプト**だけ。CDNのゲームエンジン等は基本使わない（mori系のThree.jsは例外）。
- 公開サイトは **`main` ブランチから自動デプロイ**。`main` に入って初めて反映される。

## 1. 作る前に必ずやること
1. **この畜学を読む。**
2. **作るものに近い既存ゲームを2〜3本、実際に開いて研究する**（コードの作法・操作感・演出の強さを盗む）。
   下の「既存ゲーム一覧」で**ジャンルが被らない新作**を選ぶ。
3. 共通要素（強制更新ボタン・世界ランキング・サウンド・演出）は**既存ゲームからコピーして継承**する（ゼロから書かない）。

## 2. 必ず継承する共通要素（チェックリスト）
- [ ] **🔄 強制更新ボタン**：タイトル画面（`#titleScreen` / `#startOv` / `#title`）に注入する自己完結IIFE。
      キャッシュ削除＋ServiceWorker解除＋キャッシュバスター(`?r=Date.now()`)で最新取得。**セーブ(localStorage)は消さない**。
      → `tower/` 等の最後の `<script>` をそのままコピー。
- [ ] **🏆 世界ランキング**：Firebase Realtime Database プロジェクト `chikumonogatarikiroku`、ノード `rankings/<ゲーム名>`。
      - `<head>` に firebase-app/database/auth の compat 9.23.0 を3本読み込む。
      - 匿名認証が使えれば `auth.uid`、ダメなら**端末IDにフォールバック**（必ず登録できる設計）。
      - 1端末1枠・最高スコアのみ保持（`transaction` で単調増加）。`score` ＋ サブ指標（例 `chain`/`wave`/`eggs`）。
      - UI：名前入力＋**国旗ピッカー**、結果発表演出（`.rankin-*`：ドラムロール→順位ドーン＋紙吹雪/花火、1位は👑WORLD CHAMPION）。
      - グローバルAPI `window.CoccoRank = { onGameOver(score, sub), onShowRanking(), submit(), celebrate() }`。
      - 新ノードは `docs/ranking-db-setup.md` に強化ルールと説明を追記（live は“オープン”運用でも登録は動く）。
      → `koro/` の世界ランキング `<script>` 一式＋関連CSS/HTMLをコピーして、ノード名・localStorageキー・サブ指標だけ差し替える。
- [ ] **🔊 ミュート / ⛶ 全画面 / ← もどる(`/`へ)** ボタン。
- [ ] **ベスト記録・進行は `localStorage`** に保存（ランキングだけサーバー保存）。
- [ ] **公式こっこちゃん**：`/chikumo.svg` を `Image` で読み込むか SVGスプライトで表示。色違いは「色キャンディ/バブルの上にこっこを乗せる」と読みやすい。

## 3. 中毒設計（必須）★最重要
ターゲットは **30代後半の女性とその子供**。両方が「あと1回」で止まらなくなる設計にする。
- **効果音**：WebAudioでその場生成。**コンボ/連鎖でキー(音程)が一段ずつ上がる**気持ちよさを必ず入れる。
- **BGM**：8bitチップチューンのループ。**FEVER中はテンポ＆キーUP**。
- **操作性**：スマホ最優先。`touch-action:none`、pointer/touchイベント、タップ/スワイプ/フリック。理不尽でない難度。
- **演出（全部入れる）**：パーティクル、光のリング、画面フラッシュ、画面ゆれ(shake)、`navigator.vibrate`、
  そして **FEVER（背景が虹色・得点2倍・紙吹雪）**。マイルストーン演出・カウントダウン(3,2,1,GO)も効く。
- **ループ設計**：短いプレイ→即リトライ。タイムアタックや残機＋時間ボーナスで「もう少しで…」を作る。
- すべて **Web Audio API ＋ Canvas でその場生成**、画像/音声アセットは増やさない（1ファイル完結）。

## 4. 実装の作法（house style）
- 1ファイル自己完結。インラインCSS＋インラインJS。`"use strict";` のIIFEで包む。
- Canvasは `DPR=min(devicePixelRatio,2)` でリサイズ。`requestAnimationFrame` のメインループ。
- メタ：`viewport`(`viewport-fit=cover`)、OGP、`theme-color`、apple-mobile-web-app 各種、`canonical`。
- 配色・ボタン・スクリーンは既存（koro/tower）のCSSクラス（`.screen .btn .ctrlbtn .back` 等）を踏襲。

## 5. 品質検証（「動いた」と言う前に）★今回の教訓
- 変更したHTMLの**インライン`<script>`を必ず `node --check`**（構文チェック）。
- **メインループ`frame()`は `try/catch` で包む**：1回の描画例外で `requestAnimationFrame` ループが死ぬと、以後の操作で**画面が真っ白**になる。例外でループを止めない。
- **初期状態（盤面/状態が未生成）でも `draw()` が落ちないようガードする**（空配列アクセスでの `TypeError` に注意）。盤面・HUDの描画は「プレイ中かつ生成済み」でゲートする。
- 可能なら**ヘッドレス検証**：(a) ロジック（盤面の整合・連鎖の収束・例外なし）をNodeでシミュレーション、(b) 描画ループが初期状態で生存し、開始後に要素が毎フレーム描画されることを確認。
- ❌ **実機表示を確認せずに「完成」と報告しない**。表示を伴う変更は描画まで検証する。

## 6. 新作を出すときの導線更新
- `index.html` のメニューに `<li><a href="/<dir>/">…EN…JA…</a></li>` を追加。
- `sitemap.xml` にURLを追加。
- `sw.js` の `CACHE` 版を上げる（`chikumonogatari-vNN` を +1）。
- ランキングありなら `docs/ranking-db-setup.md` にノード追記。

## 7. 公開ワークフロー（毎回・最後まで）
詳細は `release-chikumo` スキル。順番：
1. **畜史を更新**（`changelog/index.html` の `LOG` 先頭に今日のエントリ。tag: new/feat/imp/fix）
2. **構文チェック**（`node --check`）
3. **コミット＆プッシュ**（作業ブランチ）
4. **main へPR作成**（過去PRがマージ済みなら新しいPRを作る）
5. **PR番号/URLを伝えてマージをお願いする** ← ここで止める

🚫 **マージは自分で押さない**。ユーザーが明示的に「マージして」と言った時だけ `merge_pull_request`。
