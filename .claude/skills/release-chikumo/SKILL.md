# 公開ワークフロー（毎回この順で・最後まで必須）

公開サイト(<chikumo.jp>)は main ブランチから自動デプロイされる。作業ブランチへ push しただけ／
PRを作っただけでは反映されない。main にマージされて初めて公開される。

1. **畜史を更新** … changelog/index.html の `LOG` 配列の先頭に、今日の日付のエントリを追加する。
   - tag は new(新ゲーム) / feat(新機能) / imp(改善・調整) / fix(修正) から選ぶ。
   - 既に今日の日付ブロックがあれば、その items 先頭に1行足す。
2. **構文チェック** … 変更したゲームHTMLのインライン <script> を node --check で確認する。
   - 例: 該当HTMLの<script>本文を一時ファイルに書き出して `node --check 一時ファイル`。
3. **コミット & プッシュ** … 作業ブランチに対して行う。**main へ直接 push しない**。
   - 作業ブランチが無ければ作る。
4. **main へPR作成** … gh で main 宛のPRを作る。
   - 過去のPRが既にマージ済みなら、新しいコミット用に**新しいPRを作る**。
   - 同じブランチにオープンPRがあるなら、それを更新（重複作成しない）。
5. **報告して止める** … PR番号/URLを伝え、「マージはあなたが押してください」と添える。

# 厳守
- 🚫 マージは絶対に自分で押さない。`gh pr merge` 等のマージ操作はしない。
  ユーザーが明示的に「マージして」と言った時だけ。
- main へ直接コミット/プッシュしない。必ず作業ブランチ→PR。
- 畜史の追記を忘れない（公開物の履歴なので必須）。
- PR作成までを確実にやって、そこで止める。
