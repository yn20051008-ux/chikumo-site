# 畜学（ちくがく） ─ 畜物語の設計哲学

> 畜史(`changelog/index.html`)が「何を作ったか」の記録なら、畜学はその手前にある
> 「なぜ・どう作るか」の記録。**記憶を持たないClaudeでも、毎回これを読めば畜学に沿って作れる**よう、
> 哲学をコードとして外部化したもの。新しいこっこゲームを設計する前に必ず読むこと。
>
> 起源：2026-06-24、ユーザー(正論蒔宿)との対話で言語化。第一弾の実装は `kizuna/`（コッコキズナ）。

## 七つの信条

### 1. 既視感の禁止
ありきたりな既存ジャンルの焼き直しは、**もう作らない**。
（マッチ3・落ち物・もぐらたたき・タワー・ランナー…は制覇済み。それらの「こっこ版」を増やすのではない。）
目指すのは「**この世にまだ無い遊び**」の発明。ジャンル名で呼びにくいほどよい。

### 2. 中毒の「形」を選ぶ
中毒そのものは敵ではない。悪いのは**形**だ。
- ❌ 底なしの微温の中毒（ショート動画的）＝弱い快感を延々と垂れ流し、静かに人生を奪う。最もタチが悪い。
- ⭕ 畜物語が作るのは「**短く・激しく・必ず終わる**」超中毒。**60秒で焼き切る**。底（終わり）を必ず設計する。無限ループ＝底なし＝禁忌。

### 3. 「終わり」が商品
本体はプレイ中ではなく、**終わった瞬間**にある。
ドーパミンが引く瞬間に「人生に向き合う」火花が散る——その瞬間を、**設計で強制的に発火**させる。
終わりの無音・余韻こそが商品。向き合う瞬間を**罰ではなく祝福**として演出する。

### 4. 同時多発の正気
世界ランキングは順位争いの道具ではなく、**祭壇**。
世界中が同じ短い儀式を終え、**同時に顔を上げる**——その「正気の同時多発」を可視化する。
（インフラは既存Firebase `rankings/<ゲーム名>` に相乗り。`docs/ranking-db-setup.md` 参照。）

### 5. ターゲットは親子
遊び手は「親子」。一瞬こっこと遊ぶたび、**親子関係と向き合う**作品にする。
だからエフェクト・効果音・BGMは飾りではない。**その一瞬を「二人の濃い記憶」に変える本体**として全振りする。

### 6. 敵は逆縁のみ ─ ブラックこっこ
敵は**ブラックこっこ**＝自分の影＝十六式でいう**逆縁要素**だけ。
（逆縁要素＝縁を・関係を・繋がりを断ち切ろうとする「私」。）
**父でも母でも、息子でも娘でもない。唯一の敵は、自分自身のズレ。**
相手プレイヤーを攻撃対象にする要素を入れてはならない。倒すべきは常に「自分の中の切ろうとする力」。

### 7. 一ファイル自己完結
HTML＋インラインJS＋Web Audio API＋Canvas で**1ファイル完結**。
音・BGM・演出はその場生成。外部依存は世界ランキングのFirebaseのみ。
大きなインラインJSを編集したら必ず `node --check` で構文確認（→ `release-chikumo` スキル）。
