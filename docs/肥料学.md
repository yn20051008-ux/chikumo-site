# 肥料學（ひりょうがく） ─ 枯れた木を、次の芽の糧にする

> 七つの蔵：**十四式＝どう作るのか／遊戯学＝なぜ作るのか／設計学＝どの高さ・どの順で作るのか／畜史＝何を作ったのか／思考学＝どう考えたのか／肥料学＝なぜ終えたのか／技術学＝何で作るのか。**
> 畜史（`changelog/index.html`）が「成りしもの」を録し、思考学（`docs/思考学.md`）が「どう考えたか」を録すなら、
> 肥料學は**「なぜ死んだか」**を録す。木を伐るとき、理由を言葉にせず消せば、同じ根から同じ失敗がまた芽吹く。
> 枯れた木を弔うためでなく、**土に還して次の芽の糧にする**ための書。
> 公開ページは `hiryogaku/index.html`（蔵書 `/Library/` の棚）。内容を変えたら両方そろえること。
>
> 起：丙午2026・文月四、削除されたゲームの死因を記録し二度と同じ手戻りをしないための蔵として結ぶ。

---

## 巻一 ─ 標準手順（木を伐るときの再現レシピ）

**ゲームを削除・終了する（または「これは終わりにしよう」と決まる）たびに、必ずこの順で行うこと。飛ばすと、次の頭が同じ場所でまた枯らす。**

1. **伐る前に、理由を一行で言い切る** … 「なぜ終えるのか」を、ユーザーの言葉のまま・推測を混ぜずに書き出す。理由が自分でも曖昧なら、曖昧なまま消さず、まず本人に確かめる。
2. **死因を分類する** … 次のどれかに当てはめる。当てはまらない／分からないときは「分からない」と正直に書く（思考学の作法を継ぐ。証拠のない推測を教訓として残さない）。

   | 分類 | 具体例 |
   |----|----|
   | 畜学七相違反 | 壱＝既存ジャンルの焼き直し／弐＝底が無い／参＝終わりが罰／四＝順位争いの祭壇／五＝親子の的を外す／六＝相手を敵にする／七＝一器に納まらない（外部依存） |
   | 技術的死因 | 外部エンジン依存、シェーダ精度不統一などプラットフォーム起動不能、パフォーマンス、致命的バグの放置 |
   | 設計思想の継承・整理 | 前作の願い・動詞を別の木へ引き継ぐための取り下げ（焼き直しでなく発展的解消） |
   | 運用都合 | コスト・要望・その他、遊びの設計とは別の理由 |
   | 記録なし | 当時、死因を言葉にしないまま終えた（→巻二「記録の空白」を見よ） |

3. **横展開して、同じ死因を持つ現存作が無いか調べる** … これが肥料學の核心。死因を記録する目的は弔いでなく、**同じ轍を他の木・次の木で二度と踏まないこと**。同じ死因の芽（同じ外部依存、同じ精度ミス、同じジャンルの焼き直し）が他に残っていないか、実際にコードを見て確かめる。
4. **肥料學（この書）巻二に記録する** … 名前・パス・生没（分かれば）・死因の分類・具体的な症状（コードや現象があれば）・**教訓（次への一行）**・何を継いだか（後継作があれば）を書く。全部を書く書ではない──**死因が分からないなら「分からない」と書く**方が、無い理由を捏造するよりずっと安全な記録になる。
5. **畜史（changelog）に `end` タグで訃報を記す** … 遊び手への文面は既存の型（「いつも畜物語で遊んでくださり…」）を踏襲。畜史は遊び手へのお詫び、肥料學は作り手（次のAI）への教訓──**書く相手が違う。両方書く。**
6. **実体を消し、目録を編み直す** … トップ `index.html`・`sitemap.xml`・`sw.js` のキャッシュ一覧からリンク／エントリを外し、ゲーム本体のディレクトリを削除する。
7. **継として還す** … コミット＆push → mainへPR作成。**マージはユーザーに委ねる（自分で押さない）。**

## 巻二 ─ 死んだ木の記録

実際に消えた木から、何を学べるか。同じ形の死に方をしそうになったら、まずここを読む。

### 一・こっこマッチ（マッチ3 ／ `/match/`）─ 死因：既視の禁（壱）違反

- **死因**：既存ジャンル（マッチ3・キャンディクラッシュ系）の焼き直し。畜学七相の壱「既視の禁」に真っ向から反していた。記憶を持たないこっこ（Claude）が、畜学を読まずに作ってしまった一作。
- **教訓**：**新作着手前に「この世にまだ無い遊びか」を確かめる一次チェックを設計手順の先頭に固定する。** すでに思考学 巻一②（先行作の批判的レビュー）・畜史での「死んだ木」検索がこの役目を継いでいるが、根はここにある──蔵を読まずに手が動くと、真っ先にこの死に方をする。
- **継いだもの**：親子で交互にこぐ「コッコブランコ（`/buranko/`）」に作り直して置き換えた。ジャンルは捨て、置き換え先も**七相準拠から出発する**という態度だけを継いだ。

### 二・島ぐらし2（こっこの森2 ／ `/mori2/`）─ 死因：一器に納む（七）違反＋シェーダ精度不統一

- **死因**：Three.js（外部エンジン）依存で畜学七相の七「一器に納む」に違反。加えて、地形シェーダーの精度指定が頂点側 `highp`・フラグメント側 `mediump` とズレており、精度に厳格なiOSでプログラムのリンクに失敗→起動が途中で停止→「はじめから」ボタンが反応しない、という形で発覆。公開からわずか3日で取り下げた。
- **教訓**：**「一器に納む」は比喩でなく実務の制約。** 外部エンジンを使うと、その依存が想定していない環境（今回はiOSのシェーダコンパイラ）で必ずどこかが割れる。**全シェーダの `precision` は頂点・フラグメントとも `highp` に統一する**を、思考学 巻一⑦の恒久項目として組み込み済み。同じ死因の芽が他に残っていないか、WebGLを使う全作で時々見直すこと。
- **継いだもの**：外部依存ゼロ（素のWebGL1）・全シェーダ `precision highp` 統一・パスも新しく `/shima2/` として「島ぐらし2 ななつの夜明け」に建て直した（死んだ木の上には建てない）。判断の詳細は思考学 巻二を見よ。

### 三・コッコキズナ（親子の縁 ／ `/kizuna/`）─ 死因：記録なし（願いだけ継承）

- **死因**：畜史の `end` エントリはお詫び文のみで、技術的原因も七相違反も明記されていない。「作り手の都合」としか書かれておらず、後から死因を検証できない。
- **教訓**：**理由を書かずに終えると、次の頭は「同じ失敗を避けられたか」を永遠に確かめられなくなる。** これが肥料學という書そのものを作った直接の動機。今後は本手順の①②を必ず先に済ませ、理由を空白にしたまま `end` を書かない。
- **継いだもの**：「親子で息を合わせる」という願い自体は、コッコブランコ（交互の呼吸）・学習シリーズ（天秤・継・大河・地球＝親子で並んで学び直す）に、かたちを変えて受け継がれたと畜史に明記されている。**ジャンルは消えても、願いは他の木に移植できる**という例。

### 四・記録の空白 ─ ぶらんこ／積む（タワー）／振る（シェイク救出）／傾け（ジャイロ）／奥へ（ワープ）／合体（マージ）

- 畜史にはこれらすべて `end` タグの訃報が残っているが、**いずれも技術的死因・七相違反の記載が無い**（お詫び文のみ）。振る・傾け・奥へは全て「スマホを操作してZ軸の奥へ進む」という近い構造の作品が短期間に並んで終了しており、機構が近すぎたための整理だった可能性はあるが、**畜史にその判断は記録されておらず、推測を教訓として書くことはしない**。
- **教訓（この空白そのものから）**：死因を書かないまま消した木は、後から「同じ死因を持つ現存作が無いか」を横展開で確かめることができない。肥料學が生まれる前の削除には、この確認ができていない可能性がある。**新しい削除では巻一の手順を必ず踏み、この空白を今後は作らない。**

### 五・対戦（コッコランブル ／ `/battle/`）─ 死因：設計思想の継承・整理（smashへ発展的解消）

- **死因**：「smashに発展的に引き継がれたため」（ユーザーの言葉）。思考学 巻二には、後発の「コッコスマッシュ SPECIAL（`/smash/`）」がbattleの欠け（キャラ選択なし・ため無し・回避無し・ガケ無し・固定カメラ・祭壇なし・時間無制限）を埋める続編として建てられた経緯が明記されている。以降smash側だけに、つかみ・投げ・強攻撃・ジャストガード・ずらし・みだれ乱闘（1 vs 影×3のFFA）・観客の歓声・ステージ作り込みなど繰り返しレビュー・改修が積み重ねられ、battleは初版のまま更新が止まっていた。同じ「ふっとばし対戦」という林の中で、smashがbattleの完全上位互換に育った時点で、battleを閉じる判断をした。
- **教訓**：**続編を「先行の木の欠けを埋める」設計で建てた場合、続編側の改善が積み重なって前作が実質的に上位互換で置き換えられたら、両方を残したままにせず閉じる判断をする。** 焼き直しでなく発展的解消であることを畜史・肥料学の両方に明記し、次の頭が「なぜ似た2作が並んでいるのか」を推測せずに済むようにする。
- **横展開（同じ死因を持つ現存作の確認）**：`gp`/`gp2`、`rescue`/`rescue2`、`splat`/`splat3d`、`mori`/`shima2` など「無印＋続編」の組が他にも現存する。それぞれの実装・畜史・思考学の記述を確認したが、いずれも続編が前作の完全上位互換になったとは明記されておらず、`gp2`はキズナターボ等の別要素追加、`rescue2`は60秒スラッシュという別動詞、`shima2`はmori2失敗後に`mori`を存続させたまま別パスで再建、と各々独立した並存作として意図的に残されている。battle→smashと同じ死因（前作が続編の完全下位互換になり更新停止）を持つ現存作は、今回確認した範囲では見当たらなかった。
- **継いだもの**：「ふっとばし対戦」というジャンル・動詞そのものは「コッコスマッシュ SPECIAL（`/smash/`）」にすべて継承済み。世界ランキングも`rankings/smash`に一本化されている。

### 六・はじく（コッコポップ ／ `/pop/`）─ 死因：既視の禁（壱）違反

- **死因**：「既視の禁違反（既存ジャンルの焼き直し）」（ユーザーの言葉）。同じ色のこっこちゃんをタップで一掃するルールは、いわゆる同色つぶし（SameGame／Bejeweled系のタップ消去）パズルの焼き直しで、畜学七相の壱「既視の禁」に反する。
- **教訓**：**「同色を消す・つなぐ」という動詞は既に一般的なジャンルの型であり、着手前に必ず「この動詞は畜物語の他作（特に近い動詞を持つ作品）と被っていないか」を一次チェックする。** こっこマッチ（マッチ3の焼き直し）と根が同じ死因であり、「既視の禁」は繰り返し起こりうる死因として要警戒。
- **横展開（同じ死因を持つ現存作の確認）**：畜物語には「同じ色を〜」という説明を持つパズルが他にも現存する──`link`（同じ色のこっこちゃんをなぞってつなげてポン）・`bubble`（同じ色を3つ以上くっつけてパチンと弾くバブルシューター）・`koro`（同じこっこをくっつけて合体させるスイカゲーム系）。特に`link`の「同じ色をなぞってつなげてポン」は、`pop`の「同じ色をタップで一掃」と動詞レベルでかなり近く、将来同じ死因（既視の禁）を問われる可能性がある**要注意先**として記録する。今回の削除対象はユーザーが指定した`pop`のみで`link`は削除していないが、次に同ジャンルへ手を伸ばす前には必ず`link`の実装を読み直し、焼き直しでないかを再確認すること。
- **継いだもの**：特になし。ユーザーからは後継作の指定が無く、焼き直しジャンルを単純に退場させる判断だった（こっこマッチのようなジャンル置き換え作は今回無い）。

### 七・熱狂（コッコフィーバー ／ `/fever/`）─ 死因：分からない（「つまんない」）

- **死因**：ユーザーの言葉は「つまんない」のみ。七相違反・技術的死因のどれに当てはまるか確認したところ、ユーザー自身も「分からない（推測しない）」と回答した。**具体的な技術的原因も七相の何に反するかも特定できていない**ため、推測で埋めずに「分からない」と正直に書く。
- **教訓**：**「つまらない」という主観評価は、必ずしも畜学七相のどれかに機械的に対応するとは限らない。** 死因を無理に分類へ押し込めようとすると、根拠のない教訓を捏造することになる。分からないものは分からないと書く方が、次の頭にとって安全な記録になる。
- **横展開**：分類が「分からない」ため、探すべき具体的な症状（外部依存・精度不統一・特定ジャンルの焼き直し等）が無く、有効な横展開はできなかった。四章「記録の空白」と同種の限界がある記録として扱う。
- **継いだもの**：特になし。後継作の指定は無い。
