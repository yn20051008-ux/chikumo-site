## 〇 一言で

**「結んで、住まわせて、ゆるす」60〜90秒の灯まもり。**
ふわふわ漂うこっこ達を二羽ずつ縁で結び、できた家に住まわせ、家から衆→国→民→宙へと灯の格子を育てる。
ブラックこっこ（逆縁）が縁を断ち、こっこを乱域へ吹き飛ばす。住んだ家だけが放てる**赦の光**で連れ戻す。
90秒で必ず終わり、あなたが結んだ格子が夜空の星座になって浮かぶ。

## 二 畜学七相への適合（実装で守る掟）

1. **壱・既視の禁** … 後述「四 既視の禁チェック」を実装前に読むこと。**絶対に守る差別化点：こっこ同士・家同士は決して融合／消滅しない**（∿=二つのまま参照）。合体ゲー（故koro）・消しゲー（link）に寄せる実装をしない。
2. **弐・焔の形** … **90秒固定・延長なし・無限モードなし**。リトライはワンタップ。
3. **参・終を售る** … 終幕は罰でも集計画面の即出しでもない。入力停止→風の音が止む→家々の音が結んだ順にアルペジオ→格子が星座へ→**2秒の完全な無音**→静かに結果。宙到達時はオーロラが一面に流れる特別演出。
4. **四・万象同時の醒** … 世界ランキング `rankings/sumika`（既存Firebase相乗り。実装は `mebuki/index.html` のランキング部をそのまま雛形に）。記録は「最高の器（宙>民>国>衆>家）＋スコア」。
5. **五・二親の的** … **一本指だけで全操作**（親も子も遊べる）。二人同時操作は**採用しない**（肥料学で実証済みの死因）。結ばれた二羽が向かい合う姿・家に灯がともる音と光を、記憶の本体として作り込む。
6. **六・敵は逆縁のみ** … 敵はブラックこっこ（同じ品種の暗い染め）だけ。**プレイヤーはこれを攻撃できない・しない**。対抗手段は「住まわせる（住んだ家の縁は断てない）」と「赦（連れ戻す）」のみ。階層が上がる瞬間の光で影は静かに解けて退く（押し返し）。
7. **七・一器に納む** … `sumika/index.html` 1ファイル自己完結。HTML+インラインJS+Canvas2D+WebAudio。外部はGoogle Fonts（既存作と同等まで）とFirebase（ランキングのみ）。

## 三 木の仕様（実装スペック）

### 3.1 画面と領域

- 全画面Canvas2D一枚。`devicePixelRatio`対応（技術学 巻三の型）。縦横どちらでも遊べるようフィールドは正方形基準で中央配置。
- `S = min(canvas幅, canvas高さ)/2` として：
  - **循域**：中心から半径 `R_JUN = 0.62*S` の円。ほのかに明るい暖色（夜の草原に落ちる灯りのイメージ）。
  - **乱域**：`R_JUN`〜`R_HABA = 0.92*S` の帯。暗く、風の粒子が渦を巻く。
  - **絶**：`R_HABA` の外。ほぼ黒。境界の円（幅）は細い線で常時見える。
- 残り時間は**幅の円周に沿って閉じていく暁色の弧**で表す（数字のタイマーはHUD左上に小さく併記）。

### 3.3 動詞（プレイヤー操作は「なぞる」と「タップ」の二つだけ）

1. **むすぶ**：こっこAからこっこBへドラッグ（両者とも未結・距離 `0.5*S` 以内・両者とも循域内）→ ∿成立。
   二羽は向かい合い、ゆるく寄り添って漂う。足元にゴースト🏠が出る。+10点。
   - 距離超過や対象なしでリリース → 糸がほどけるだけの柔らかい失敗演出（減点なし・音も咎めない）。
2. **すまわせる**：ゴースト🏠をタップ → 住。二羽が家に収まり（家の左右にちょこんと座る）、灯がともる。+20点。
   - 住んだ家は**動かなくなり**、糸は**断てなくなる**。
3. **くらいをあげる**：住んだ家 → 住んだ家 へドラッグ（両方住・距離 `0.7*S` 以内・どちらも未昇格）→ 衆🏘️が二軒の上に浮かぶ。+50点。
   同様に 衆→衆で国🏯(+150)、国→国で民🏙️(+400)、民→民で**宙🌌(+1000)**。
   - 昇格の瞬間、光の波紋が広がり、**画面内のブラックこっこは全員ほどけて退く**（六の押し返し。撃破ではない）。
4. **ゆるす（赦）**：住んだ家 → 乱域のこっこ へドラッグ → 赦の光が伸び、0.7秒で循域内（その家の近く）へ連れ戻す。+30点。
   - **住んだ家からしか撃てない**（赦の実行は住から）。ゴースト家・独りのこっこからは不可。
   - **乱域にいる間だけ有効**。幅の外（絶）へ出たものは対象外（廿式の規則そのまま）。

※ 3つのドラッグは同じ一本指ジェスチャ。始点が「未結こっこ／住んだ家」かで自動判別する。判別不能な曖昧ケースは「近い方を優先」でよいが、**必ず予告線の色で何が起きるかを描き分ける**（むすぶ=白金、くらい=暖金、赦=金＋粒子）。

### 3.4 逆縁（ブラックこっこ）の振る舞い

- 初出現 `t=12秒`。以後の出現間隔は 9秒 → 終盤4秒 まで線形に短縮。同時最大3体。
- 乱域からふらりと入り、**最も近い「未住の糸」**へ歩く（歩速 `0.06*S/秒`）。糸に触れて**1.2秒**接触し続けると断つ：
  - 予兆：糸が震え、低い唸り音（この1.2秒が「急いで住まわせる」カウンターの猶予）。
  - 断たれると：∿がぷつりと切れ（悲しい単音）、二羽は**乱域へ放り出され**、ゴースト家は消える。
- 乱域のこっこは外向きにゆっくり流され、**約5秒**で幅の縁に達し、絶（星屑✨→少し後に中央へ新しい認が湧く）。
- 逆縁は**プレイヤー・住んだ家・昇格済みの家には何もできない**。プレイヤーからも一切攻撃できない。

### 3.5 スポーンと数値（初期値。検証プレイで調整可、調整したら本文書に追記）

- 開始時：こっこ6羽が循域内にランダム配置（フェードイン）。
- 追加スポーン：2.0秒ごとに1羽（中央付近に認として）。フィールド上の**未結こっこが8羽**を超えている間は湧かない。
- 90秒での理論上の宙到達：結16+住16+昇格15＝47ジェスチャ＋防衛。1ジェスチャ約1.2秒として約70秒＝**達人がぎりぎり届く夢の頂**。普通の子は衆〜国、上手い親で民、が想定着地。

### 3.6 スコアと祭壇（ランキング）

- スコア＝上記の加点合計。結果画面は「最高の器の絵文字（🌌🏙️🏯🏘️🏠）＋スコア＋結んだ縁の数＋ゆるした数」。
- 世界ランキング：`rankings/sumika`。**ソートキーは `rank*100000 + score`**（rank: 宙5 民4 国3 衆2 家1 なし0）で「どの器まで育てたか」を主役にする。実装・名前入力UI・Firebase非同期ローダーは **mebukiのランキング部を丸ごと雛形**にする（新規DB設定不要。詳細は `docs/ranking-db-setup.md`）。
- ベストスコアはlocalStorage（キー `sumika_best` 等。小さく使う＝祖元学 巻二）。

### 3.7 音（WebAudio合成のみ。外部音声ファイル禁止）

- **世界が育つほど音楽が育つ**構造にする（五・記憶の本体）：
  - 住んだ家1軒ごとにペンタトニック（C-D-E-G-A系）の持続音が一声ずつ加わり、環境和音が厚くなる。
  - むすぶ＝短いプラック（結ぶたび僅かに音高が上がる）／住＝あたたかい鐘＋和音入り／昇格＝器が上がるほど太いファンファーレ（でも上品に短く）／赦＝ハープのグリッサンド／断＝弦がほどける単音／絶＝小さな鈴一つ。
  - 逆縁接近＝低い唸り（近いほど僅かに大きく）。
- 終幕：全持続音がすっと引く→結んだ順のアルペジオ→**完全な無音2秒**→結果。
- 音の番人：`pagehide`でAudioContextをclose（思考学 巻一⑨）。

## 四 既視の禁チェック（壱）─ 実装で寄せてはいけない近傍

設計時に現存作・故人・一般ジャンルと突き合わせ済み。**実装がこの差分を崩したら壱違反になる**：

| 近傍 | 骨格の違い（守るべき一線) |
|----|----|
| link（現存/なぞって消す） | 本作は**何も消さない**。なぞった結果が残り、育ち、守る対象になる。マッチ（同種を揃える）要素を入れない |
| koro（故/合体・スイカ系＝壱違反で死亡） | **融合演出を絶対に作らない**。∿は「二つのまま参照」。昇格も二つの家が「残ったまま」上に器が浮かぶ |
| rescue2（現存/なぞって斬る） | 本作のドラッグは切断でなく接続。攻撃動詞ゼロ |
| タワーディフェンス一般 | 砲台・攻撃・敵HPが無い。防衛手段は「住まわせる」と「赦」＝building+forgivenessで、敵を殺す骨格ではない |
| 電力網・パイプ接続パズル一般 | 静的な正解配置を探すパズルではない。リアルタイムで揺れ動く実体を結び、断たれ、連れ戻す動的な世話 |
| あやとり・ひも結びおもちゃ | 物理ひもシミュではない。糸は縁の記号 |

「結んで住まわせ、赦しで守り、器の階層を登る」という当たり判定の骨格に、既存ジャンル名は付かない（付けにくいほど良い＝壱）。

## 六 技術指定（技術学 巻三〜巻五に従う）

- **Canvas2D一枚**＋`requestAnimationFrame`ループ（dtは可変・0.05秒でクランプ）。WebGL禁止ではなく「不要だから使わない」。
- パーティクル・糸・こっこは配列管理＋毎フレーム全走査で足りる規模（最大: こっこ約40・糸約20・粒子数百）。
- 入力はpointerイベントで統一（PC/スマホ両対応）。`touch-action:none`。
- こっこ描画：起動時に `/chikumo.svg` をロードしてオフスクリーンCanvasにアトラス（通常向き・反転・暗色ティントの3種を焼く）。ロード完了までの一瞬だけ丸のフォールバック可（畜学の補のとおり仮表示に留める）。
- **検証フック `window.__sumika` を常設**（思考学 巻一⑧。先例 `__mebuki`）：
  `state()`（phase・残秒・こっこ/糸/家/各器の数・乱域の数・スコア）／`tscale(n)`（時間倍率）／
  `linkAt(i,j)`・`dwellAt(k)`・`promoteAt(a,b)`・`graceAt(h,c)`（各動詞をプログラムから発火）／
  `shadowNow()`（逆縁を即出現）／`skipToEnd()`（終幕へ）。
- インラインJS編集後は必ず `node --check`（scriptを抜き出して確認）。

## 七 実装手順（この順で。飛ばすと手戻る＝設計学）

1. 骨格：Canvas・ループ・領域3円・タイマー弧・タイトル/プレイ/結果のphase遷移。
2. こっこ：svgアトラス→スポーン→浮遊→認フェード。
3. 動詞1「むすぶ」＋∿描画（描画と判定の共通点列）＋ゴースト家。
4. 動詞2「すまわせる」＋灯の演出＋和音一声。
5. 逆縁：出現→糸へ歩く→予兆→断→乱域へ放出→外流→絶（星屑→新しい認）。
6. 動詞3「ゆるす」。
7. 動詞4「くらいをあげる」＋階層の器＋押し返し波紋。
8. 音の全体（3.7）とHUD・あそびかた・🔄ボタン。

10. 検証（下記八）→数値調整→調整値を本文書3.5に追記。
