# 畜物語サイト（chikumo.jp）

絵文字の島でのんびり遊ぶ、こっこちゃんのブラウザゲーム集。各ゲームは `<ディレクトリ>/index.html`
にHTML+インラインJS+WebGL等で自己完結（例: `mori/`=こっこの森, `gp/`=こっこGP, `splat3d/` ほか）。
更新履歴は `changelog/index.html`（通称「畜史」）。

## 🧭 畜学（設計哲学）─ 新ゲームを作る前に必ず読む

**新しいこっこゲームを設計・実装する前に、必ず次の七つを読むこと。七つの蔵の役割：**
**十四式（`jushishiki/index.html`）＝どう作るのか／遊戯学（`yugigaku/index.html`＝七相の秘伝）＝なぜ作るのか／設計学（`sekkeigaku/index.html`）＝どの高さ・どの順で作るのか／畜史（`changelog/`）＝何を作ったのか／思考学（正典 `docs/思考学.md`・公開 `shikogaku/index.html`）＝どう考えたのか（判断が分かれた場所の記録と標準手順＝軽量AIでも再現できる思考の再現レシピ）／肥料学（正典 `docs/肥料学.md`・公開 `hiryogaku/index.html`）＝なぜ終えたのか（削除したゲームの死因記録＝枯れた木を次の芽の糧にする）／技術学（正典 `docs/技術学.md`・公開 `gijutugaku/index.html`）＝何で作るのか（グラフィック・音・保存に使う技術と使わない技術の記録＝軽量AIでも実装を再現できるレシピ）。**
**思考学の掟：新しい木を建てたら、迷った場所・間違えた場所・二分して確かめた場所を `docs/思考学.md` 巻二に追記して還すこと。**
**肥料学の掟：ゲームを削除・終了するときは、必ず先に「なぜ終えるのか」を理由の推測なしで言い切り、死因を分類し、同じ死因を持つ現存作が無いか調べてから `docs/肥料学.md`（および `hiryogaku/index.html`）巻二に追記して還すこと。→ 詳細は「🗑️ ゲーム削除時の必須手順」を見よ。**
**技術学の掟：グラフィック・音・保存で新しい技法を使ったら、または既存作の実装を調べて発見があったら `docs/技術学.md`（および `gijutugaku/index.html`）に追記して還すこと。**

⚠️ **設計は「森→林→木」の順（高きより降りる）。木から積むな。**
設計学（正典 `docs/設計学.md`／公開 `sekkeigaku/index.html`）の核：設計には高さがある。
**森（全体＝十四式の家⑧）→ 林（束＝縁⑦）→ 木（一＝衡⑥）** の順に、高きより降りて作る。
森（世界・掟・共有の臓器）を定めぬまま林や木を作ると、後で森に合わず**抜いて植え直す＝手戻り**になる
（衡＝0 なら縁は発らず、家＝森にならない）。木で得た差分は継として森へ還し、地図（CLAUDE.md・docs）を編み直す。
❌ 過去の失敗：森を設計しないまま林・木を設計して手戻りした（戒め）。→ **必ず森から降りよ。**

⚠️ **十四式を「ラベル」として扱うな。必ず開いて読み、深さを理解せよ。**
十四式は構造実体の成立・遷移・停止を記す固定文法（認→…→縁→家→…→継、継→認で周回）。
中心は ⑦ **縁＝衡＋ ∿ 衡−**（二つの均衡が出会わぬまま並んで参照しあう波）。**逆縁とは、その ∿ を断つ力**。
こっこの戯びはここから降りてくる ── 例：コッコキズナ＝親子（衡＋/衡−）の縁を、逆縁（ブラックこっこ）が断とうとするのを押し返す遊び。
「読める」と「理解できる」は違う。読むだけで素通りした過去がある（戒め）。土台を理解してから設計せよ。

設計哲学の七つの信条（畜学）は正典 `docs/畜学.md`（末尾に平語の運用の鍵）、公開の秘伝は `yugigaku/index.html`・`chikugaku/index.html`（蔵書 `/Library/` 内）。守りは暗号でなく深さに委ねる（平文で公開、理解できる者だけが受け取る）。要点だけ先に：

1. **既視感の禁止** … 既存ジャンルの焼き直しはもう作らない。「この世にまだ無い遊び」を発明する。
2. **中毒の形** … 底なしの微温（ショート動画的）は禁忌。「短く・激しく・必ず終わる」超中毒（60秒で焼き切る／無限ループ禁止）。
3. **終わりが商品** … ドーパミンが引く＝人生に向き合う瞬間を設計で発火。終わりの無音・余韻を「罰でなく祝福」として演出。
4. **同時多発の正気** … 世界ランキングは順位争いでなく、世界が同時に顔を上げる祭壇。
5. **ターゲットは親子** … 親子で遊ぶたび親子関係と向き合う。音・光・BGMは飾りでなく「二人の記憶を濃くする本体」。
6. **敵は逆縁のみ** … 敵はブラックこっこ＝自分の影＝十六式の逆縁要素（縁を切ろうとする「私」）だけ。相手（親/子）を敵にしない。
7. **一ファイル自己完結** … HTML＋インラインJS＋WebAudio＋Canvasで1ファイル。世界ランキングだけFirebase相乗り。

## 🚨 公開ワークフロー（毎回・最後まで必須）

**公開サイト(chikumo.jp)は `main` ブランチから自動デプロイされる。**
作業ブランチに push しただけ／PRを作っただけでは**反映されない**。`main` にマージして初めて公開される。

変更が一段落したら、毎回この順でやること（→ 詳細は `release-chikumo` スキル）：

1. **畜史を更新** … `changelog/index.html` の `LOG` 先頭に今日のエントリを追加（tag: new/feat/imp/fix）
2. **構文チェック** … 変更したゲームHTMLのインライン`<script>`を `node --check`
3. **コミット & プッシュ**（作業ブランチ）
4. **main へPR作成**（過去PRがマージ済みなら、新コミット用に**新しいPRを作る**）
5. **PR番号/URLを伝えて、マージをお願いする** ← ここで止める

🚫 **マージは絶対に自分で押さない。マージボタンはユーザーが押す。**
`merge_pull_request` は使わない（ユーザーが明示的に「マージして」と言った時だけ）。
PR作成までを確実にやり、「マージはあなたが押してください」と添えて報告する。
（ユーザーがマージした後、確認を頼まれたら `git fetch origin main && git rev-list --count origin/main..HEAD` が 0 かで検証する）

❌ 過去にやった失敗：PR作成だけで「反映済み」と誤認／既存PRマージ後の新コミットを放置／畜史更新忘れ／**勝手にマージした**。
→ **PR作成まで＋畜史更新を確実に。公開（マージ）はユーザーの操作。**

## 🗑️ ゲーム削除時の必須手順（消す前に必ず読む）

**ゲームを削除・終了する（「もう終わりにしよう」と決まる）たびに、消す前に必ず死因を考えて記録すること。目的は弔いでなく、同じ失敗・同じ手戻りを二度と起こさないこと。**

1. **伐る前に、理由を一行で言い切る** … 「なぜ終えるのか」を、ユーザーの言葉のまま・推測を混ぜずに書き出す。曖昧なら消さず先にユーザーへ確かめる。
2. **死因を分類する** … 畜学七相違反／技術的死因（外部依存・精度・パフォーマンス等）／設計思想の継承・整理／運用都合／記録なし、のどれか。当てはまらない・分からないなら**「分からない」と正直に書く**（無い理由を捏造しない）。
3. **横展開する** … 同じ死因を持つ現存作が無いか、実際にコードを見て確かめる。**ここが目的そのもの＝同じ轍を他の木で二度と踏まないこと。**
4. **`docs/肥料学.md`（公開 `hiryogaku/index.html`）巻二に記録する** … 名前・パス・死因・具体的な症状・**教訓（次への一行）**・何を継いだか（後継作があれば）。両方のファイルをそろえて更新する。
5. **畜史（changelog）に `end` タグで訃報を記す** … 遊び手への文面（既存の型「いつも畜物語で遊んでくださり…」を踏襲）。畜史は遊び手へのお詫び、肥料学は次のAIへの教訓── **書く相手が違う。両方書く。**
6. **実体を消し、目録を編み直す** … トップ `index.html`・`sitemap.xml`・`sw.js` からリンク／エントリを外し、ゲーム本体のディレクトリを削除する。
7. **公開ワークフロー（🚨節）どおりコミット・push・PR作成**。マージはユーザーに委ねる（自分で押さない）。

❌ 過去の失敗（肥料学を作った動機）：技術的死因の記録なしにお詫び文だけで終えたゲームが複数あり（コッコキズナ・ぶらんこ・積む・振る・傾け・奥へ・合体）、後から「同じ死因が他に残っていないか」を確かめられなくなっている。
→ **今後の削除では必ず①②③④を先に済ませ、この空白を二度と作らない。**

## 補足
- ゲームHTMLは1ファイル自己完結。大きなインラインJSを編集したら必ず `node --check` で構文確認。
- 機能追加=`feat`、調整・強化=`imp` で畜史を分けると履歴が読みやすい。
